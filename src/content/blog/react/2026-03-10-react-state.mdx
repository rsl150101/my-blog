이전 포스트에서 React의 선언적 UI 개념을 살펴봤다. 하지만 단순히 선언만 한다고 해서 화면이 저절로 바뀌지 않는다. 이번 글에서는 React의 진짜 핵심인 <strong>상태(State)</strong>와 화면을 다시 그리는 **리렌더링**의 원리를 파헤쳐보자.

### 1. 리렌더링: React는 언제 화면을 다시 그릴까?
React에서 화면이 업데이트되는 과정을 <strong>리렌더링(Re-rendering)</strong>이라고 한다. 이전 글에서 단순히 변수를 수정한다고 해서 화면이 바뀌지 않는 이유는, React가 해당 변수의 변화를 감시하지 않기 때문이다.
- **동작 원리:** React는 <strong>상태(State)</strong>가 변경되거나, 전달 받은 <strong>속성(props)</strong>이 갱신되거나, 구독 중인 <strong>컨텍스트(Context)</strong>의 값이 변했을 때 리렌더링을 트리거한다. 이때 **해당 컴포넌트를 포함한 모든 하위 자식 컴포넌트들 또한 연쇄적**으로 리렌더링한다.
- **수동 렌더링의 한계:** 값이 바뀔 때마다 `render()` 함수를 직접 호출해 렌더링할 수도 있지만, UI가 복잡해지면 어떤 데이터가 변했을 때 어떤 함수를 실행해야 할지 추적하기가 어렵다.
<details>
<summary>수동 렌더링</summary>

```jsx
    const root = document.getElementById("root");
    let counter = 0;

    const render = () => ReactDOM.render(<Container />, root);
    const Span = () => <span id="totalClick">Total clicks : {counter}</span>;

    const Btn = () => (
      <button
        onClick={() => {
          counter += 1;
          render();
        }}
      >
        Click me
      </button>
    );

    const Container = () => (
      <div>
        <Span />
        <Btn />
      </div>
    );

    render();
```
</details>

### 2. 성능의 핵심: 가상 DOM(Virtual DOM)
Vanilla JS는 데이터가 바뀌면 HTML 전체를 새로 갈아 끼우는 경향이 있어 성능이 떨어지기 쉽다. 반면, React는 가상 DOM이라는 복사본을 활용한다.
1. **메모리에 가상 트리 생성:** 실제 화면에 그리기 전, 메모리 상에서 먼저 UI를 구성한다.
2. **비교(Diffing):** 이전 상태의 가상 DOM과 새로 바뀐 가상 DOM을 비교한다.
3. **최소 업데이트:** 진짜 바뀐 부분(예: 숫자만 0에서 1로)만 찾아내어 실제 브라우저 DOM에 딱 그 부분만 반영한다.

컴포넌트가 너무 많아 이 비교 과정 조차 부담이 될 때가 있다. 그래서 `React.memo`나 지연 로딩(Lazy loading) 같은 도구로 불필요한 비교 조차 최적화하는 과정을 거친다.

### 3. 상태 관리: `useState`
React에서 상태를 만들고 업데이트할 때는 `useState` Hook을 사용한다.

`const [state, setState] = useState(initialValue);`

React에게 데이터가 변했으니 화면을 다시 그리라고 알리는 방법은 `setState` 함수를 호출하는 것이다. `setState`로 상태를 업데이트 하는 방법은 두 가지 방법이 있다.
- **직접 할당(`setState(newValue)`):** newValue로 값을 설정한다. 이전 상태 값에 의존하지 않고 단순한 업데이트에 좋다.
- **함수형 할당(`setState((prevValue)=> !prevValue)`):** 권장되는 방식이다. React의 업데이트는 비동적으로 일어날 수 있는데, 이 방식을 쓰면 React가 가장 최신의 상태 값(`prevValue`)을 보장해주기 떄문에 이전 상태 값을 안전하게 참조할 수 있다.

### 4. 실습: 단위 변환기 앱
```jsx
const root = ReactDOM.createRoot(document.getElementById("root"));

const App = () => {
  const [amount, setAmount] = React.useState("");
  const [flipped, setFlipped] = React.useState(false);
  const handleAmountChange = (event) => {
    setAmount(event.target.value);
  };
  const reset = () => {
    setAmount("");
  };
  const flip = () => {
    reset();
    setFlipped((current) => !current);
  };

  return (
    <div>
      <div>
        <h1>Time Converter</h1>
        <label htmlFor="minutes">Minutes</label>
        <input
          value={flipped ? amount * 60 : amount}
          placeholder="Minutes"
          type="number"
          id="minutes"
          onChange={handleAmountChange}
          disabled={flipped}
        />
      </div>
      <div>
        <label htmlFor="hours">Hours</label>
        <input
          value={flipped ? amount : Math.round(amount / 60)}
          placeholder="Hours"
          type="number"
          id="hours"
          onChange={handleAmountChange}
          disabled={!flipped}
        />
      </div>
      <button onClick={reset}>Reset</button>
      <button onClick={flip}>Flip</button>
    </div>
  );
};

root.render(<App />);
```
- Input 태그를 다룰 때는 React 상태와 입력 값을 동기화하는 **제어된 컴포넌트** 방식이 권장된다.
- **HTML 속성 주의:** JSX에서 `for`은 `htmlFor`로, `class`는 `className`으로 작성해야 한다.
- **단일 루트 규칙:** JSX는 반드시 하나의 부모 요소 또는 `<> </>` (Fragment)로 감싸야 한다. React가 어떤 DOM 트리를 렌더링할지 정확히 인식하기 위한 규칙이다.

### 5. 제어된 컴포넌트란?
- **비제어 컴포넌트:** `<input>`의 값을 브라우저가 스스로 들고 있는 상태.
- **제어된 컴포넌트:** `<input>`의 `value`를 React의 `state`와 연결한 상태.

비제어 컴포넌트를 사용한 코드는 React는 그 값을 읽어오기 전까지는 무엇이 입력됐는지 모른다. <br/>
제어된 컴포넌트를 사용한 코드는 사용자가 입력을 하면 -> `onChange` 이벤트 발생 -> `setState`로 상태 업데이트 -> 다시 `input`의 `value`로 전달한다.
#### 왜 이렇게 하나요?
입력 값이 실시간으로 React의 통제 하에 있기 때문에, 입력된 즉시 단위를 변환하거나, 유효성 검사를 하거나, 리셋 버튼으로 값을 초기화하기가 매우 쉬워진다.

### 6. 마치며
React는 단순히 코드를 짧게 만들어주는 도구가 아니라, **데이터(State)가 변하면 UI는 알아서 바뀐다**는 철학을 구현한 라이브러리다. 효율적인 리렌더링을 위해 컴포넌트 패턴과 상태 관리의 원리를 이해하는 것이 중요하다.
